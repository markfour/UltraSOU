Welcome to JINS MEME SDK

To help you to download JINS MEME SDK and start programming your application(s) which interacts with JINS MEME, the following guidance provides you with the overview of SDK and how it works.
You won't need to understand the specifications of Bluetooth 4.0 completely, yet some specific BLE words are found in this document.
JINS MEME SDK requires CoreBluetooth framework. Please add this framework to your project on XCode.

About Application Authorization

Both App ID and App Secret are required to develop your own application(s). Please create your JINS account and create app to get your APP ID and App Secret.
Once you've got your App ID and App Secret, please pass them to "setAppClientId method of MEMELib" as parameters. By doing so, validation of your SDK and authorization will be completed. Without this authorization, MEMELib won't work.

DataMode in JINS MEME

JINS MEME has 2 data modes: Standard Mode and Realtime Mode.
In Standard Mode, JINS MEME sends data once per minute for lower power consumption. This mode is optimized for longer-time data analyses rather than realtime interactive applications.
In Realtime Mode, JINS MEME sends data at roughly 26Hz. This mode suits applications that require “instant” data acquisition and analyses while increasing JINS MEME power consumption instead.
The application in Realtime Mode needs to stay on foreground, and data mode can be changed by "MEMELib's changeDataMode".

About data storage in JINS MEME

JINS MEME has a function to store data of Standard Mode in its local storage. Even if data can not be sent to user's smartphone for some reasons, Standard Mode data for 16 hours can be stored in its storage.
Below three are the cases.
• When the data mode is in Realtime.
• While JINS MEME is on, it is not connected to the application of user's smartphone.
• The application is not working correctly.
Once JINS MEME has recovered from the reasons above, JINS MEME restarts to send the stored data in FIFO ( First In / First Out ) order.
