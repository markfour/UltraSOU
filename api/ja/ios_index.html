JINS MEME SDKへようこそ

このドキュメントには、開発者がはじめてJINS MEME SDKをダウンロードして、JINS MEMEと連動するアプリケーションを開発するために、必要な情報を記述しています。
Bluetooth 4.0（BLE）の仕様を理解していなくても開発は進められますが、開発ドキュメントの一部にはBLE特有の用語が使用されています。
なお、JINS MEME SDKはCoreBluetooth frameworkを使用しています。XCode上でプロジェクトに追加してください。

アプリケーション認証について

JINS MEMEのアプリケーションを開発、配布するためにはアプリID / アプリSecretを取得する必要があります。JINSアカウントを作成して、アプリを作成してください。
取得したアプリID / アプリSecretをMEMELibのsetAppClientIdの引数に入れて認証をおこなってください。この認証をおこなわない限り、MEMELibは動作しません。

データモードについて

JINS MEMEにはスタンダードモードとリアルタイムモードの２つのデータモードがあります。
スタンダードモードは、JINS MEMEの消費電力を抑えて1分間に１回の通信を行います。比較的長いスパンでのデータ解析を行うようなアプリケーション開発を目的としています。
一方、リアルタイムモードは、データの送出頻度が高いため、消費電力は増えますが、瞬目（瞬き）やそれに付随する情報を約26Hzで取得することが可能です。
データモードはMEMELibのchangeDataModeで変更できますが、アプリケーションがフォアグラウンドにある必要があります。

JINS MEME内でのデータストアについて

JINS MEMEはスタンダードモードのデータをデバイス内に貯める機能があります。なんらかの原因によってJINS MEME からアプリケーションにスタンダードモードのデータが渡っていないと判断されるとJINS MEMEはそのデータをデバイス内メモリに最大１６時間分保存します。
データが渡っていないと判断されるのは下記の３ケースです。
・データモードがリアルタイムモードに設定されている
・JINS MEMEの電源は入っているが、アプリケーションと接続されていない場合
・アプリケーションがなんらかの理由で正常動作していない場合
JINS MEME内に溜まったデータはその原因が解消されると、すぐにFIFO ( First In / First Out )スタイルでアプリケーションへと送信されます。
